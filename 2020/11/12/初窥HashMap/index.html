初窥HashMap

发表于2020-11-12|更新于2021-01-07|JavaContainer
|字数总计:3k|阅读时长:12分钟|阅读量:|评论数:

本文基于JDK1.8的HashMap源码分析

前言

  1. 什么是HashMap？
    • HashMap是基于哈希表(散列表)，实现Map接口的双列集合。
    • 数据结构是“链表散列”，也就是数组+链表 。
    • 使用key-value的方式存取数据。key是唯一的，value可以重复。
    • 允许存储null 键null 值。
    • 元素无序。
  2. HashMap 的数据结构？
    • 哈希表结构（链表散列：数组+链表）实现，结合数组和链表的优点。
    • 当链表长度超过 8 时，链表转换为红黑树。

HashMap的构造方法

Put方法

存储对象时，将 key和value键值传给 put() 方法：（put方法再调用putVal()方法）

  1. 判断table是否为null或者长度为0。如果是，则对table进行扩容。
  2. 根据key的hash和table长度计算出数组下标（注：hash(key){return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);}）
  3. 判断数组对应位置是否为空。如果为空，则直接创建一个节点放入对应的位置。
  4. 如果不为空
    1. 先判断当前位置的node节点是否就是包含key的节点
    2. 如果不是包含key的节点，再判断当前节点是不是红黑树的节点，如果是则通过红黑树来存入key和value
    3. 如果不是红黑树的节，则肯定是链表的节点。
      1. 遍历链表，找到对应的包含key的节点
      2. 如果没有找到，则尾插法添加一个新节点
      3. 在遍历的末尾的时候，也会判断当前链表的节点是否超过的指定的阈值（TREEIFY_THRESHOLD）
      4. 如果超过，则链表转换为红黑树
  5. 判断是否找到包含key的节点。如果找到，则根据条件进行value替换
  6. 如果没有找到
    1. ++modCount;
    2. 判断是否需要扩容。需要则扩容.

流程写的可能不是很简洁，完美，但是这是我自己第一次看源码做的的分析。如果后续还会复习，再做简化。

源码分析：

扩容

  1. 如果当前表已经初始化了，则判断容量是否已经到达了最大值
    1. 如果到了最大值，则把阈值调整到Integer.MAX_VALUE，并不在对表进行扩容
    2. 如果没有，则进行扩容(newCap = oldCap << 1)，并判断是否扩容前的容量达到了默认容量，达到了，则扩容的同时，阈值也要提高(newThr = oldThr << 1;)。
  2. 如果当前表还没有初始化，则进行初始化扩容。
    1. 如果创建HashMap的时候指定了初始容量，则扩容为指定的容量(构造方法会自动地将指定的容量转换为 2 的 n 次方)
    2. 如果没有指定初始容量，则扩容为默认的初始容量static final int DEFAULT_INITIAL_CAPACITY = 1 << 4;
  3. 确定好了扩容后的容量和阈值后，则开始扩容。
  4. 创建一个新的散列表(Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];)，并直接使用(table = newTab;)。
  5. 如果旧表中没有数据，则扩容完成。
  6. 如果旧表中有数据，则遍历每一个节点添加到新表中。（(e.hash & oldCap) == 0 表示索引位置不变。否则，新下标=扩容前下标+扩容前容量）

关于数组容量

HashMap 构造函数允许用户传入的容量不是 2 的 n 次方，因为它可以自动地将传入的容量转换为 2 的 n 次方。

HashMap的工作原理

HashMap底层是hash数组和单向链表实现，数组中的每一个元素都是链表。由Node内部类(实现Map.Entry接口)实现，HashMap通过put&get方法存储和获取。

存储对象时，将K/V键值传给put()方法：

  1. 如果当前table还没初始化，则先初始化扩容

  2. 调用hash(key)方法计算K的hash值，然后结合数组长度，计算得到数组下标；

  3. 判断当前key的hash值是否再HashMap中

    1. 如果不在，则执行插入
    2. 如果在，且key的equals返回true，则更新键值对
    3. 如果在，且key的equals返回false，则插入链表尾部(1.7是头插法)

获取对象时，将K传给get()方法：

  1. 调用Hash(key)方法，结合数组长度获取该键值所在链表的下标
  2. 遍历链表，通过equals()方法找到相同的Node节点，获取K对应的V值
